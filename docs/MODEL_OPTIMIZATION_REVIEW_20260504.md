## 二、关键发现与诊断

### 2.1 5日模型 - F1=0.9168 的真实含义

**现象分析：**
- F1从35.17%激增至91.68%，涨幅超160%
- 全量模式(17699只标的)触发，单实验运行
- 使用 `event_adaptive_label_q18` 标签方案（自适应标签±1%中性区）
- 657,855个训练样本（全方向标签）

**潜在风险：**
1. **验证集时间切分可能偏向** - 由于全量模式下分割方式为时间序列切分（80%-20%），验证集样本可能与训练期分布接近
2. **标签漂移问题** - 自适应标签量化区间(±1%)随市场波动调整，回测期可能存在"看后镜"效应
3. **样本失衡** - 657K样本中涨跌幅分布不均，F1高可能掩盖Precision或Recall的倾斜
4. **过度拟合信号** - Brier=0.125虽合理，但与F1=0.9168的巨大差异暗示模型在某些阈值范围内过度优化

**建议验证：**
- 对比单独数据周期验证结果（仅用2026Q2数据），检测泛化能力
- 输出 Precision-Recall 曲线，而非仅看F1

---

### 2.2 20日模型 - F1=0.5008，样本筛选影响

**现象分析：**
- 从17699只标的筛选至618只（过滤率96.5%）
- 样本量从657K降至93K
- 单周期多实验网格搜索（通常3-5个参数组合）
- 标签方案同样是 `event_adaptive_label_q18` 但参数调优不同

**问题根源：**
1. **标的严格筛选** - 中期模型要求历史数据充分(20+120天)，自动过滤新股/停牌/数据缺失
2. **样本构造复杂性** - 20日窗口下，样本独立性假设减弱，标签滞后效应更明显
3. **特征有效性下降** - 中期特征（20日MA、MACD等）的区分度低于短期（5日RSI、ATR）
4. **参数搜索空间小** - 仅618个标的，参数优化可能陷入局部最优

**实际表现：**
- AUC=0.682（高于50%随机线），证明有一定区分能力
- F1=0.5008恰好突破最低门槛(0.50)，风险边际

---

### 2.3 60日模型 - F1=0.5468，最长周期平衡

**现象分析：**
- 筛选至579只标的（过滤率98.85%）
- 样本72,751个
- AUC=0.7239（三者最高），但F1=0.5468

**特点:**
- **AUC反而最优** - 在概率排序上表现最佳，说明长期趋势预测相对稳定
- **F1略低但更稳定** - 可能反映长期预测的非线性性，阈值选择影响大
- **样本质量最高** - 经过最严格筛选，参数空间充足

**风险：**
- AUC高但F1低的矛盾，暗示阈值（threshold）的选择离优点远
- 可能需要调整概率校准策略

## 三、系统层面的问题

### 3.1 训练框架问题

| 问题 | 表现 | 影响 |
|------|------|------|
| **全量模式单实验** | 5日仅运行1个实验 | 无法对比不同参数组合，可能遗漏最优解 |
| **样本构建耗时** | 5日需75分钟，20日需22分钟 | 迭代周期长，难以频繁优化 |
| **标签方案固定** | 三个周期都用event_adaptive_label_q18 | 没有考虑周期特异性的标签设计 |
| **阈值搜索简化** | F1最大化 | 未考虑实际业务成本（假正/假负），可能不是最优决策 |

### 3.2 验证方法问题

- **仅用验证集F1指标** - 缺少Precision、Recall、ROC曲线
- **无K折交叉验证** - 验证结果对某一时间切分敏感
- **缺少样本外测试** - 未在真实预测期（如未来5天）验证模型
- **无WFB评估集成** - 上阶段提及WFB评估，未见融合到训练流程

## 四、模型优化建议

### 优先级 P0 - 验证基准（1-2天）

**P0-1: 实现5日模型的双路验证**
```
操作：
1. 切分验证策略：
   - 方案A（现状）: 时间序列80%-20%
   - 方案B（严格）: 2026Q2单月数据（仅用Q1训练）
   
2. 增加指标输出：
   - Precision, Recall, F1各自
   - ROC-AUC + 曲线图
   - 混淆矩阵 (TP/FP/TN/FN)
   
3. 输出对标：当前F1=0.9168 vs Q2严格验证下的实际F1
```

**P0-2: 实现阈值选择的业务化**
```
操作：
改进 _find_best_threshold，从纯F1最大化改为复合评分：
  - score = w_profit * (precision-recall平衡) + w_auc * auc - w_cost * fpr
  - 其中 w_profit=0.5, w_auc=0.3, w_cost=0.2（可调）
  - 这样避免过度优化F1，兼顾实际收益
```

---

### 优先级 P1 - 参数优化（3-5天）

**P1-1: 为全量5日模型启用参数网格搜索**
```
当前: ≥1000只标的时仅跑 1个实验（event_adaptive_label_q18#1）

改进方案：
1. 在全量模式下启用 3个精选实验：
   - event_adaptive_label_q18（现有）
   - fp_guard_q24 或 q26（guard模型，防止极端）
   - 一个中性标签方案（对标），作为对照组
   
2. 每个实验跑 3-5个参数组合（XGBoost depth, learning_rate 调优）

3. 并行化：改用多进程加速模型fit，预期可减少20-30%耗时
```

**P1-2: 中期(20日)和长期(60日)的标签方案分化**
```
20日专用标签设计：
  - 中性区不用固定1%，改用浮动 0.8%-1.5%（基于20日ATR）
  - lookback_years 改为 0.8-1.0（比5日短）
  - 原因：20日周期中局部反转更常见，需更灵活的中性区

60日专用标签设计：
  - 中性区 1.2%-2.0%（更宽松，减少噪声）
  - lookback_years 改为 1.5-2.0（更长历史）
  - 原因：长期趋势更稳定，需过滤短期波动
```

---

### 优先级 P2 - 特征工程（5-10天）

**P2-1: 增加周期特定的特征**
```
5日短期：
  - 日内成交量分布（开盘买卖比）
  - 5分钟级别的微观结构特征（如果数据可得）
  - 近3日的情绪反转指标

20日中期：
  - 20日内机构净买卖额变化率
  - 融资余额与股价的乖离率
  - 行业轮动指标（相对强度）

60日长期：
  - 基本面更新周期（财报期接近度）
  - 宏观因子（CPI/PMI的动量）
  - 产业景气度指数
```

**P2-2: 特征重要性排序与清洗**
```
1. 输出每个实验的特征重要性 TOP-20
2. 移除相关性>0.85的冗余特征对
3. 检测特征缺失率，剔除缺失>30%的特征
```

---

### 优先级 P3 - 集成与融合（2周以上）

**P3-1: 模型融合框架**
```
当前：三个周期模型独立
目标：实现加权融合预测

方案：
  short_5d_prob * w_short + medium_20d_prob * w_medium + long_60d_prob * w_long
  其中权重 w 基于最近30天各模型的精度加权

示例（假设近期表现）：
  w_short=0.5（AUC=0.66, 但反应快）
  w_medium=0.25（AUC=0.68, 次稳定）
  w_long=0.25（AUC=0.72, 最稳定但滞后）
```

**P3-2: 动态阈值调整**
```
基于市场状态自动调整预测阈值：
  - 低波动期（VIX替代品）: 提高阈值，降低假正
  - 高波动期: 降低阈值，抓住机会
  - 缩小版的regime switching
```
