#### **优化4：回测维度增强**
```
原有复盘指标：
│
├─ target_hit (是否达到目标价)
├─ max_profit_rate (最高盈利%)
└─ days_to_target (多少天达到)

新增复盘指标：
│
├─ max_drawdown (最大回撤%) ← 捕捉风险
│  └─ 示例: -8.3% 表示持仓中最多亏8.3%
│
└─ excess_return_vs_hs300 (超额收益%) ← 衡量alpha
   └─ 示例: +5.2% 表示相较基准指数多赚5.2%
```

**投资决策改进**：
- 不仅看"是否涨"，还看"亏多少"和"赚多少"
- 例：两个都达到目标的建议，但一个max_drawdown=-3%，一个-12%
  → 选前者，因为持仓体验更好

**定量预期**: 
- 能筛选出"收益高风险低"的建议
- 可能发现某些高命中率但高回撤的建议需要改进

## 📞 问题快排

**Q: 为什么Recall会下降？**  
A: 强中性区（±1.5%/2.0%）会过滤掉涨幅在-1.5%~+1.5%之间的"模棱两可"样本。这导致标签为"1"的样本减少，模型对"涨"的样本接触变少，recall下降是正常的。这是precision-recall的权衡，总体AUC往往会改善。

**Q: PMI硬编码50.0是否合理？**  
A: 50.0是PMI荣枯线（景气线），>50表示扩张，<50表示收缩。当无法获得实时PMI时，用中性值50.0是保守的、不引入偏见的做法。

**Q: Walk-forward需要多久？**  
A: 每个窗口都是一次完整训练，6窗口共需约6×80min=480min≈8小时。可选执行（非必需），用于特殊场景验证模型稳定性。

**Q: 如果结果不理想怎么办？**  
A: 模型框架本身是合理的。若指标回落，可能是：
1. 数据特性变化（需重训）
2. 中性区设置过度（试试±1.2%）
3. 宏观特征兜底值不当（调整PMI/CPI值）
