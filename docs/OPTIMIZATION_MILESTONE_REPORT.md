#### Step 14: 回测验证系统
**文件**: `reviews/backtest_validator.py` (420行, 新建) + `api/reviews.py` (+75行)

**功能**:

| 指标 | 说明 | 计算逻辑 |
|------|------|---------|
| **止盈准确率** | 建议目标价被实际达成的比例 | 最高价 ≥ 目标价 |
| **加仓盈利率** | 加仓后30天内的平均收益率 | (当前价 - 建议价) / 建议价 * 100% |
| **整体准确率** | 两者加权平均 | (TP准确率 + 加仓盈利率) / 2 |
| **AI建议** | 自动判断 | ≥70% → 系统表现良好; <50% → 谨慎跟随 |

**API 端点**: `GET /api/reviews/backtest-report?days_lookback=30`  
**实际测试**: ✅ HTTP 200, 完整分析数据返回

## 🔄 系统现在的工作流

### 日常工作流

```
每日 09:30 (市场开盘后)
└─ 晨间信号告警
   ├─ 重算5/20/60日概率 (市场感知)
   ├─ 生成未来信号 (持仓风险 + 操作建议 + 推荐)
   └─ 推送告警 (邮件/微信)

每日 16:00 (收盘后)
└─ 收盘信号总结
   ├─ 回顾日内信号执行情况
   ├─ 更新操作建议优先级
   └─ 推送日终总结

每日 21:30 (市场闭市后)
└─ 3日样本增量回填
   ├─ 采集最新3天的历史数据
   ├─ 更新概率样本库
   └─ 为明日预测准备数据

每周日 04:00 (周期深度校准)
└─ 完整60日重校准
   ├─ 重新计算所有参数
   ├─ 检测参数漂移
   ├─ 调整市场感知度
   └─ 生成健康报告 (如果等级下降→告警)
```

### 用户查询流

```
用户访问持仓页面
   ↓
加载 /api/holdings/future-signals
   ├─ 查询当前持仓信息
   ├─ 获取最新推荐+预测
   ├─ 计算风险+目标+建议
   └─ 返回信号数据
   ↓
前端渲染仪表板 (3个信号卡:红/黄/绿)
   ├─ 风险预警 (红色): 显示严重/中等风险
   ├─ 操作建议 (黄色): 止盈/加仓信号
   └─ 推荐资产 (绿色): Top 5候选资产

用户点击"生成回测报告"
   ↓
查询 /api/reviews/backtest-report
   ├─ 遍历历史持仓
   ├─ 验证止盈目标达成率
   ├─ 验证加仓建议盈利率
   └─ 返回组合准确率+AI建议
   ↓
前端展示回测结果表格
```
