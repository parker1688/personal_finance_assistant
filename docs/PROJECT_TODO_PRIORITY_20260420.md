# 项目完整优化待办清单（2026-04-20）

## 总体目标
围绕以下四个方向持续推进：

1. 安全加固
2. 稳定性与 Bug 修复
3. 性能优化
4. 模型与产品能力增强

---

## P0：必须优先处理

### 1. 管理接口统一鉴权与授权
- 为训练、补采、配置修改、日志删除等接口增加管理员权限控制
- 增加角色区分：管理员 / 操作员 / 只读用户
- 防止未授权触发高负载任务

涉及模块：
- `api/model.py`
- `api/backfill.py`
- `api/routes.py`
- `api/logs.py`

### 2. 模型导入与反序列化安全加固
- 限制上传文件类型、大小、扩展名
- 防止路径穿越
- 对模型文件增加签名或可信校验
- 降低恶意 pickle 风险

涉及模块：
- `api/model.py`

### 3. 关键管理操作增加审计日志
- 记录谁触发了训练、补采、配置修改、模型导入
- 记录时间、参数、执行结果、来源 IP
- 为后续排障和安全追踪提供依据

---

## P1：高优先级优化

### 4. 预测写入与推荐写入做幂等化
- 防止重复预测记录
- 防止重复统计导致复盘失真
- 增加唯一约束与 upsert 机制

涉及模块：
- `scheduler.py`

### 5. 统一模型质量门槛判断
- 训练、上线、监控、推荐、复盘使用同一套口径
- 避免“监控说可用，推荐又降级”的不一致

涉及模块：
- `predictors/model_manager.py`
- `api/reviews.py`
- `api/recommendations.py`

### 6. 持续优化训练性能
- 滑窗训练替代全历史重复扫描
- 特征缓存
- 增量训练
- 减少长时间 CPU 占用

涉及模块：
- `predictors/model_trainer.py`

### 7. 统一外部数据源重试、限流、熔断
- 为采集器补统一 timeout / retry / backoff
- 接口失败时自动降级到缓存或备用数据源
- 防止采集拖慢系统

涉及模块：
- `collectors/stock_collector.py`
- `collectors/fund_collector.py`
- `collectors/news_collector.py`

### 8. 数据库会话与异常处理标准化
- 逐步减少泛化异常吞没
- 统一 session 使用方式
- 提高故障定位效率，减少锁表和连接泄漏风险

涉及模块：
- `api/dashboard.py`
- `api/model.py`
- `reviews/reviewer.py`

---

## P2：中优先级增强

### 9. 增加整链路集成测试
- 补充 调度 → 采集 → 训练 → 推荐 → 复盘 的完整测试
- 提前发现运行态问题，而不是只靠单元测试

涉及模块：
- `tests/test_predictors.py`
- 新增 integration tests

### 10. 提升监控与可观测性
- 增加任务运行历史
- 训练失败原因可视化
- 采集源健康状态面板
- 自动优化效果趋势图

涉及模块：
- `templates/model_monitor.html`
- `api/model.py`
- `api/logs.py`

### 11. 统一输入校验与分页限额
- 所有接口参数做 schema 校验
- 对 limit / page size 设置上限
- 避免恶意请求拉高负载

---

## P3：中长期优化

### 12. 数据文件 schema 与版本治理
- 统一 CSV 字段格式
- 明确日期格式
- 加载时做校验和兼容迁移

涉及模块：
- `data/validators.py`

### 13. 配置与部署安全收紧
- 去掉弱默认值
- 生产环境必须显式配置关键参数
- 强化安全响应头和 CSRF 防护

涉及模块：
- `config.py`
- `app.py`

### 14. 文档与运维规范补全
- 故障排查文档
- 训练策略说明
- 审计与安全运维说明
- 部署 / 回滚手册

---
